# Explicación Completa de la Aplicación NodoCivico

## ¿Qué es NodoCivico?

Es una aplicación Android para reportar problemas comunitarios (baches, luces dañadas,
basura, etc.). Permite a los ciudadanos crear reportes, ver el estado de los mismos, y
gestionar su perfil.

## Cómo Funciona la Aplicación

### 1. Navegación entre Vistas (Navigation Component + NavHostFragment)

La navegación se configura en nav_graph.xml:

SplashFragment → LoginFragment → HomeFragment
                                        ├── ReportListFragment
                                        ├── CreateReportFragment
                                        ├── ProfileFragment
                                        └── SettingsFragment

¿Cómo se comunican las vistas?
- MainActivity.java contiene el NavHostFragment que carga el gráfico de navegación
- Los fragmentos usan Navigation.findNavController(view).navigate(R.id.action_...)
  para navegar
- ViewBinding conecta los layouts XML con el código Java

Ejemplo en HomeFragment.java (líneas 37-52):
```java
binding.cardReports.setOnClickListener(v ->
    Navigation.findNavController(view).navigate(R.id.action_homeFragment_to_reportListFragment));
```

### 2. Ingreso a la App (Usuario y Contraseña) - LoginFragment.java

El sistema actual valida localmente (NO conecta a un servidor):

Validaciones que hace:
1. Campos vacíos → Muestra error "El correo/contraseña es requerido"
2. Formato de email → Debe ser válido (contiene @ y dominio)
3. Longitud de contraseña → Mínimo 6 caracteres

Código (líneas 39-73):
```java
private void validateLogin(View view) {
    String email = binding.etEmail.getText().toString().trim();
    String password = binding.etPassword.getText().toString().trim();

    // Validar campos vacíos
    if (TextUtils.isEmpty(email)) {
        binding.tilEmail.setError("El correo es requerido");
        return;
    }

    // Validar formato email
    if (!android.util.Patterns.EMAIL_ADDRESS.matcher(email).matches()) {
        binding.tilEmail.setError("Ingrese un correo válido");
        return;
    }

    // Validar contraseña mínima
    if (password.length() < 6) {
        binding.tilPassword.setError("La contraseña debe tener al menos 6 caracteres");
        return;
    }

    // Login exitoso → navegar a Home
    Toast.makeText(requireContext(), "Bienvenido a Nodo Cívico", Toast.LENGTH_SHORT).show();
    Navigation.findNavController(view).navigate(R.id.action_loginFragment_to_homeFragment);
}
```

Nota importante: El login actual es falso - acepta cualquier email válido con contraseña de
6+ caracteres.

### 5. Estados de los Reportes

Los reportes pueden tener 3 estados (definidos en Report.java línea 13 y ReportAdapter.java):

| Estado      | Color (#F57C00) | Significado                    |
|-------------|-----------------|--------------------------------|
| Pendiente   | Naranja         | Reporte nuevo, sin atender     |
| En proceso  | Azul            | Estan trabajando en ello       |
| Resuelto    | Verde           | Ya se solucionó                |

Prioridades:

| Prioridad | Color        | Significado   |
|-----------|--------------|---------------|
| Alta      | Rojo         | Urgente       |
| Media     | Amarillo     | Normal        |
| Baja      | Verde        | Puede esperar |

Ejemplo de datos en ReportListFragment (líneas 55-59):
```java
reportList.add(new Report(1, "Bache en calle principal", "...", "Infraestructura",
"Alta", "Pendiente"));
reportList.add(new Report(2, "Luz callejera dañada", "...", "Servicios Públicos",
"Media", "En proceso"));
reportList.add(new Report(3, "Basura acumulada", "...", "Limpieza", "Baja", "Resuelto"));
```

### 6. Lista de Reportes - ReportListFragment.java + ReportAdapter.java

Usa RecyclerView para mostrar la lista de reportes.

Flujo:
1. ReportListFragment crea una lista de objetos Report
2. Pasa la lista al ReportAdapter
3. El Adapter usa onBindViewHolder() para renderizar cada item
4. Cada item muestra: título, descripción, categoría, prioridad y estado

Código del Adapter (líneas 61-78):
```java
public void bind(Report report) {
    binding.tvReportTitle.setText(report.getTitle());
    binding.tvReportCategory.setText(report.getCategory());
    binding.tvReportPriority.setText(report.getPriority());
    binding.tvReportStatus.setText(report.getStatus());

    // Aplica colores según estado y prioridad
    int statusColor = getStatusColor(report.getStatus());
    binding.tvReportStatus.setTextColor(statusColor);

    int priorityColor = getPriorityColor(report.getPriority());
    binding.tvReportPriority.setTextColor(priorityColor);
}
```

### 7. Guardado de Datos

Actualmente NO usa Room Database. Usa datos simulados (mock).

ReportRepository.java (líneas 14-56):
```java
public class ReportRepository {
    private static ReportRepository instance;
    private final MutableLiveData<List<Report>> reportsLiveData;
    private List<Report> cachedReports;

    // Singleton
    public static synchronized ReportRepository getInstance() {
        if (instance == null) {
            instance = new ReportRepository();
        }
        return instance;
    }

    // Obtener todos los reportes (simulado)
    public LiveData<List<Report>> getAllReports() {
        loadMockReports();  // Carga datos de ejemplo
        reportsLiveData.setValue(cachedReports);
        return reportsLiveData;
    }

    // Crear reporte (simulado)
    public void createReport(Report report, OnReportCreatedListener listener) {
        new android.os.Handler().postDelayed(() -> {
            report.setId(cachedReports.size() + 1);
            report.setStatus("Pendiente");  // ← Estado inicial
            cachedReports.add(report);
            reportsLiveData.postValue(cachedReports);
            if (listener != null) {
                listener.onSuccess(report);
            }
        }, 500);
    }
}
```

Qué falta para Room:
1. Agregar dependencia de Room en build.gradle
2. Crear clase AppDatabase con @Database
3. Crear DAOs con @Dao
4. Reemplazar cachedReports con operaciones de Room

### 8. Comunicación con Backend (preparado para API REST)

Ya está configurado Retrofit para conectar con un servidor:

ApiClient.java:
```java
private static final String BASE_URL = "https://api.nodocivico.com/";

public static Retrofit getClient() {
    retrofit = new Retrofit.Builder()
            .baseUrl(BASE_URL)
            .addConverterFactory(GsonConverterFactory.create())
            .client(getOkHttpClient())
            .build();
    return retrofit;
}
```

ApiService.java (endpoints definidos):
```java
// Login
@POST("api/login")
```
